Building a Text Viewer Component

April 13, 2026

I shipped a text viewer that can handle files with millions of lines without breaking the browser. Here's what I learned.

The Problem

Users needed to browse huge log files and text dumps. Loading everything into the DOM would freeze the browser. We needed something that could handle arbitrary file sizes smoothly, with no libraries, no dependencies.

Virtual Scrolling

The answer is virtual scrolling. You only render what's visible, plus a small buffer. As the user scrolls, you dynamically update what's in the DOM. Constant memory, smooth performance, no matter the file size.

The algorithm is simple:

  1. Calculate visible lines based on scroll position
  2. Render only those lines plus a buffer
  3. Update on scroll events

Implementation

You need three pieces of information:

  • Line height — pixels per line
  • Viewport height — visible space
  • Total lines — file size

Here's the core logic:

function getVisibleRange(scrollTop, lineHeight, viewportHeight, totalLines) {
  const visibleLines = Math.ceil(viewportHeight / lineHeight);
  const startLine = Math.floor(scrollTop / lineHeight);
  const buffer = 5;

  const start = Math.max(0, startLine - buffer);
  const end = Math.min(totalLines, startLine + visibleLines + buffer);

  return { start, end };
}

Then render only those lines:

function render(lines, { start, end }, startLine) {
  const offsetPixels = startLine * lineHeight;

  const html = lines
    .slice(start, end)
    .map((line, i) => `
${line}
`)
    .join('');

  container.style.transform = `translateY(${offsetPixels}px)`;
  container.innerHTML = html;
}

Results

A 10 million line file renders instantly. Scroll is smooth (60 fps). Memory usage stays constant. No frameworks, no bloat.

Lessons

Understand the fundamentals. React-window and react-virtualized exist, but understanding how virtual scrolling works made it easy to build exactly what we needed.

Keep it simple. We didn't need pagination, advanced search, or fancy UI. Just a viewer that doesn't break. Ship that, add more later if needed.

Measure real data. In production, some files had variable line heights due to long lines wrapping. We adjusted the algorithm to handle that. Testing with real data caught issues theory wouldn't.

Ship It

This shipped in production a month after we started. It works. People use it daily. No regrets.

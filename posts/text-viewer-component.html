Building a Text Viewer Component

April 13, 2026

I recently built a lightweight text viewer component for a project that needed to display large files without breaking performance. Here's how I approached it and what I learned.

The Problem

We were displaying text files that could be hundreds of thousands of lines long. Loading the entire file into the DOM wasn't an option—it would freeze the browser. We needed a solution that could handle large files smoothly while remaining lightweight and responsive.

The core challenge: how do you display an arbitrary portion of a large text file without loading the whole thing?

The Approach: Virtual Scrolling

Virtual scrolling is a technique where you only render the visible portion of content in the DOM, plus a small buffer above and below. As the user scrolls, you dynamically update what's rendered. This keeps the DOM lightweight and performance snappy.

The basic algorithm:

  1. Calculate how many lines fit in the visible viewport
  2. Track the scroll position to determine which lines should be visible
  3. Render only those lines (plus a buffer)
  4. Update on scroll events

Implementation Details

The component needs to know three things:

  • Line height: In pixels, so we can calculate scrollable space
  • Viewport height: How many pixels we have to work with
  • Total lines: The full size of the file

Here's the core rendering logic:

function getVisibleRange(scrollTop, lineHeight, viewportHeight, totalLines) {
  const visibleLines = Math.ceil(viewportHeight / lineHeight);
  const startLine = Math.floor(scrollTop / lineHeight);
  const bufferLines = 5; // Render extra lines as buffer

  const start = Math.max(0, startLine - bufferLines);
  const end = Math.min(totalLines, startLine + visibleLines + bufferLines);

  return { start, end };
}

Then, for rendering:

function renderVisibleLines(lines, range, startLine) {
  const offsetPixels = startLine * lineHeight;

  const rendered = lines
    .slice(range.start, range.end)
    .map((line, i) => createLineElement(line, range.start + i));

  container.style.transform = `translateY(${offsetPixels}px)`;
  container.innerHTML = rendered.join('');
}

Performance Wins

This approach scales beautifully. With virtual scrolling:

  • A 1 million line file renders in milliseconds
  • Scroll is buttery smooth (60 fps)
  • Memory usage stays constant regardless of file size
  • No external dependencies needed

Edge Cases and Learnings

Variable line heights: If lines have different heights (wrapping text, large content), the math gets harder. For our use case, we assumed monospace with fixed line height. If you need variable heights, you'd need a more complex approach.

Search and navigation: Users want to jump to specific lines or search. We added a simple line number input that calculates the scroll position and jumps directly there.

Keyboard navigation: Arrow keys, Page Up/Down, and Home/End all work via the scroll position. It's just math.

Conclusion

Virtual scrolling is a powerful pattern for displaying large datasets efficiently. It's not new—libraries like react-window use it—but understanding the fundamentals helped us build something lightweight and tailored to our specific needs.

The best part? No dependencies. Just vanilla JavaScript and a bit of math. Sometimes the simplest solution is the right one.
